Trade-offs

Pros and cons of adding Kong to your setup

Kong is a real piece of infrastructure. It solves genuine problems but also adds cost and complexity. Here's the honest picture.

Pros

✅
One endpoint for every tool. VS Code, CLI, Obsidian, custom scripts — they all point to localhost:8000. No more switching tabs or re-pasting context.
✅
Automatic quota management. Kong tracks daily usage per provider in shared memory and silently fails over when a limit approaches. You never see a quota error mid-session.
✅
Change models in one place. New Gemini version ships? Edit one line in kong/kong.yml, restart Kong. Every tool is upgraded instantly.
✅
API key security. Keys live in Kong (or Key Vault on Azure). No client ever holds a key. If a script leaks, there's nothing to steal.
✅
Rate limiting built-in. Prevents a runaway script from exhausting your monthly OpenRouter budget in an hour.
✅
Cost optimisation. Simple prompts route to cheap models. Complex ones escalate. Estimated 60–80% reduction in OpenRouter spend vs. always using the premium model.
✅
Vendor independence. Swap any provider in 5 minutes by editing the Kong config. No code changes in any client.
✅
Request logging. Every call is logged with which model was used, tokens consumed, and routing reason. Full auditability.

Cons

⚠️
Extra infrastructure cost. ~$10–40/month on Fly.io or Azure. On top of your $40 in subscriptions, that's $50–80/mo total before any per-token LLM spend.
⚠️
New single point of failure. If Kong crashes, every client breaks. Local Docker-based Kong avoids this for personal use.
⚠️
Added latency. ~5–15ms per request (see the Latency section). Negligible against LLM response times but worth understanding.
⚠️
Claude Pro and Gemini Advanced do NOT give API access. These web subscriptions can't be routed through Kong. You need separate API keys — which are pay-per-token (see Subscriptions section below).
⚠️
Config maintenance. Model strings go stale. Providers change their API formats. Kong YAML needs occasional updates.
⚠️
Keyword classifier is simple. A heuristic Lua script misclassifies some prompts. A sophisticated routing service would need a separate ML classifier (extra cost and latency).
⚠️
Overkill for pure personal use. If you only use one tool (Claude.ai or Gemini.google.com) manually, Kong adds complexity with no benefit.
Latency

How much does Kong add?

Short answer: 5–15ms of Kong overhead on requests that take 300ms–30,000ms to complete. That's 0.05% to 5% overhead — generally imperceptible.

End-to-end breakdown (one request)

Client → Kong (local Docker) <1 ms loopback, negligible
Client → Kong (Fly.io, same region) 5–15 ms single network hop via Fly's Anycast
Client → Kong (Azure, same region) 5–20 ms Azure Container Apps ingress
Kong Lua pre-function (classify prompt) <1 ms in-process, pure Lua regex, no I/O
Kong → Smart Router (internal network) 1–3 ms same datacenter, private network
Smart Router → OpenAI (gpt-4o-mini) TTFB 300–600 ms time-to-first-byte, fast model
Smart Router → Anthropic (claude-sonnet) TTFB 500–1,500 ms time-to-first-byte, heavier model
Smart Router → OpenRouter (kimi / deepseek) TTFB 400–900 ms varies by provider load
Total streaming response (simple, 200 tokens) 1–3 sec gpt-4o-mini streams fast
Total streaming response (complex, 500 tokens) 5–20 sec claude-sonnet, longer output

What DOES affect latency

The factors that actually matter for perceived speed:

FactorImpactYour control
Which model is chosen gpt-4o-mini is 3–5× faster than Claude Sonnet High — Kong's classifier decides this
Output token length More tokens = longer streaming time Medium — set max_tokens in request
Streaming vs. non-streaming Streaming feels 3–10× faster (first token faster) High — set "stream": true
Provider load / time of day Peak hours add 200–800ms Low — Kong can retry on timeout
Your geographic distance from provider Cross-continent adds 100–300ms Medium — deploy Kong in nearest region
Kong geographic placement Local Docker = 0ms; cloud = 5–20ms High — run Kong locally for personal use
Tip: For personal use, run Kong locally with Docker (docker compose up). This eliminates the network hop entirely — Kong overhead drops to <2ms and you pay nothing for cloud hosting.
Critical Clarification

Your subscription plans and API access are different products

This is the most important thing to understand before building your Kong setup. The $20/mo you pay to Claude and Gemini buys you web UI access — not API credits. Kong routes API calls, not web sessions.

Claude Pro ($20/mo) and Gemini Advanced ($19.99/mo) cannot be routed through Kong. These subscriptions give you access to the web interface at claude.ai and gemini.google.com. They do not include API access. Kong is an API gateway — it routes API calls only.

$20/mo — Claude Pro

Claude Pro gives you access to Claude.ai — the web interface, Projects, extended context, and priority access. It is a consumer subscription, not an API product.

  • Separate account at console.anthropic.com
  • Pay-per-token billing (no flat fee)

  • Opus 4: ~$15 / 1M input tokens
  • Independent of your Pro subscription

$19.99/mo — Google One AI Premium

Google One AI Premium gives you Gemini Advanced access in the Gemini web app and inside Google Workspace (Docs, Gmail, etc). Like Claude Pro, it's a web subscription.

  • Gemini Advanced at gemini.google.com

Practical Strategies

Three ways to run this setup for your actual workflow

Now that you know the subscriptions vs. API distinction, here's how to structure your setup depending on how you work.

A

Keep subscriptions for manual work, use Kong for programmatic work

Continue using claude.ai and gemini.google.com directly for your daily chat, code review, and document work — you already paid for those. Point Kong at the free Google AI Studio API + OpenRouter for all programmatic use: CLI tools, IDE plugins, scripts, automation.

This gives you the best of both worlds: full-featured web UIs for interactive sessions, cheap/free API access for automated workflows.

API cost: ~$0–5/mo (Google free tier + minimal OpenRouter)
B

Cancel Claude Pro, redirect $20 to the Anthropic API

If you use Claude primarily through code tools (VS Code, Continue.dev, terminal) rather than the web UI, you likely get more value from $20/mo in API credits than from the Claude Pro subscription.

$20/mo on the Anthropic API at claude-sonnet-4-5 pricing (~$3/1M input tokens) buys roughly 6.6 million input tokens/month — far more than Claude Pro's soft daily cap. Route through Kong to use every dollar efficiently.

API cost: $20/mo Anthropic API + free Google AI Studio + OpenRouter overflow
C

Full consolidation: cancel both subscriptions, go API-only

If your usage is mostly API-driven (IDE plugins, automation, scripts), cancel both subscriptions and put the full $40/mo into a mix of: Anthropic API, Google AI Studio paid tier, and OpenRouter (Kimi 2.7, DeepSeek Flash v4).

Kong manages the budget across all providers. You get the latest models, smart routing, and full programmatic control — for roughly the same $40/mo total spend.

API cost: ~$40/mo split across Anthropic + Google + OpenRouter via Kong

How to point a tool at Kong

Any OpenAI-compatible tool can be redirected to Kong by changing its base URL:

# Python (openai SDK)
from openai import OpenAI
client = OpenAI(
    base_url="http://localhost:8000/v1",
    api_key="anything"  # Kong ignores this; it injects the real key
)
response = client.chat.completions.create(
    model="auto",          # Kong picks the model
    messages=[{"role": "user", "content": "Hello"}]
)

# Aider CLI
OPENAI_API_BASE=http://localhost:8000/v1 aider --model auto

# Continue.dev config.json
{
  "models": [{
    "title": "Smart-Cost (Kong)",
    "provider": "openai",
    "model": "auto",
    "apiBase": "http://localhost:8000/v1",
    "apiKey": "kong"
  }]
}
Recommendation

The practical setup for your situation

Given you pay $20/mo to Gemini and $20/mo to Claude and want to run all models simultaneously without hitting limits, here's the most pragmatic approach:

1
Keep both web subscriptions — use them for manual sessions

claude.ai and gemini.google.com are excellent products. Keep using them directly for conversational work, document editing, and anything where the web UI adds value (file uploads, projects, shared threads).

2
Get a free Google AI Studio API key — today

Go to aistudio.google.com, create a project, generate an API key. The free tier gives you Gemini 2.0 Flash at 1,500 requests/day — plenty for personal programmatic use. This is separate from your $20/mo subscription and costs nothing.

3
Add $5–10/mo to Anthropic API for API-based Claude access

Create an account at console.anthropic.com and add $10 in API credits. This gives you ~3.3 million tokens on claude-sonnet-4-5 — enough for heavy coding sessions. Your Claude Pro web subscription continues to exist separately.

4
Add an OpenRouter key for Kimi 2.7 and DeepSeek

Create an account at openrouter.ai and add $5–10. OpenRouter is pay-per-token with no monthly minimum. Use it for overflow, long-context tasks (Kimi), and fast/cheap completions (DeepSeek). At DeepSeek's pricing ($0.14/1M tokens), $5 buys ~35 million tokens.

5
Run Kong locally with Docker — no cloud cost

docker compose up on your laptop. Zero latency overhead. Zero cloud hosting cost. All three services (Kong, smart-router, frontend) run locally. When your laptop is on, every tool that points at localhost:8000 gets smart multi-model routing automatically.
