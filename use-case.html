The Problem

Three things go wrong without a gateway

Running multiple subscriptions manually creates real operational headaches.

🚫

You hit Gemini or Claude's daily limit mid-day — and don't know until it fails

Both Gemini Advanced and Claude Pro have soft usage caps that reset daily. Heavy use in the morning — long coding sessions, big document analysis — can exhaust the quota by noon. Without monitoring, the next request just errors out. You waste time switching tabs, logging into the other service, and re-pasting context.

💸

You're paying for subscriptions but defaulting to OpenRouter anyway

If you're not actively tracking usage, you unconsciously route too much traffic to OpenRouter (pay-per-token) even when you have headroom left on your flat-rate subscriptions. You lose the value of the $40/month you already paid.

🔄

"Latest model" is a moving target — you're always chasing it manually

Google ships a new Gemini version. Anthropic releases Claude Sonnet 4.5, then Opus 4. Moonshot updates Kimi. DeepSeek ships a faster Flash variant. Without a central config, you're updating model strings scattered across tools, scripts, and chats individually.

🧩

No single interface — context switching kills flow

Four models means four browser tabs or four CLI configs. Every switch breaks your train of thought. You want one endpoint that hides the complexity.

How Kong Fixes It

Kong acts as a smart quota manager and model router

Every request goes to one Kong endpoint. Kong tracks quota consumption per provider, picks the best available model, injects the right API key, and falls back gracefully when a subscription limit is close — all before a single token is generated.

Step by Step

What Kong does on each request

1

Request arrives at Kong — one endpoint for everything

Any tool — a CLI, VS Code extension, Obsidian plugin, custom script — sends POST http://localhost:8000/v1/chat/completions with a standard OpenAI JSON body. The client sets "model": "auto" and lets Kong decide the rest.

2

Kong reads today's quota counters from shared memory

The pre-function Lua plugin calls ngx.shared.quota_counters:get("gemini_today") and ngx.shared.quota_counters:get("claude_today"). These are in-process counters that reset at midnight via a Kong timer.

3

Prompt is analysed for special routing signals

Beyond quota, Kong checks: Is the prompt multimodal (image bytes in the body)? Is it longer than 100K characters (needs Kimi's long context)? Does it contain coding/debugging keywords that warrant Claude Opus? These override the default quota-based path.

4

Kong sets provider headers and rewrites the model field

Based on the decision, Kong sets X-Provider, X-Model, and injects the correct Authorization header (Google API key, Anthropic key, or OpenRouter key) from Kong's secret store — no key ever touches the client. It also rewrites the model field in the request body to the exact model string the provider expects.

5

Kong increments the provider's daily counter

After routing headers are set, the plugin does ngx.shared.quota_counters:incr("gemini_today", 1) so the next request has accurate data.

6

Response comes back in standard OpenAI format

The smart-router service normalises Gemini, Claude, and OpenRouter responses into a single OpenAI-compatible schema. The client receives choices[0].message.content regardless of which model answered, plus an x_routing field showing which provider was used and why.

Staying Current

How Kong keeps you on the latest models

New model versions ship every few weeks. Without a gateway, you update strings everywhere. With Kong, you update one YAML file and restart. 

# kong/kong.yml — update one place to upgrade every client simultaneously
x-models: &models
  gemini_primary:   "gemini-2.5-pro-preview-06-05"  # ← update here
  claude_primary:   "claude-sonnet-4-5-20251001"     # ← update here

The Kong Config

Core routing logic in Kong's Lua plugin

This runs on every request, sub-millisecond, before any LLM token is generated.

-- Quota thresholds
local GEMINI_LIMIT  = 1500   -- approx daily cap on $20 plan
local CLAUDE_LIMIT  = 1000
local WARN_THRESHOLD = 0.80  -- start switching at 80%

-- Read today's counters from shared memory
local counters = ngx.shared.quota_counters
local g_used = counters:get("gemini_today") or 0
local c_used = counters:get("claude_today") or 0

local g_pct = g_used / GEMINI_LIMIT
local c_pct = c_used / CLAUDE_LIMIT

-- Check for special routing signals
local is_long_ctx  = #prompt > 100000
local is_multimodal = body.messages and has_image(body.messages)
local needs_opus   = prompt:lower():find("deep analysis") or
                      prompt:lower():find("think carefully")

-- Decision tree
local provider, model
if is_long_ctx then
  provider, model = "openrouter", MODELS.kimi_overflow
elseif is_multimodal then
  provider, model = "gemini",     MODELS.gemini_primary
elseif needs_opus and c_pct < WARN_THRESHOLD then
  provider, model = "anthropic",  MODELS.claude_powerful
elseif g_pct < WARN_THRESHOLD then
  provider, model = "gemini",     MODELS.gemini_primary
elseif c_pct < WARN_THRESHOLD then
  provider, model = "anthropic",  MODELS.claude_primary
elseif g_pct < 1.0 or c_pct < 1.0 then
  provider, model = "openrouter", MODELS.kimi_overflow
else
  provider, model = "openrouter", MODELS.deepseek_fallback
end

-- Set headers for smart-router to act on
kong.service.request.set_header("X-Provider", provider)
kong.service.request.set_header("X-Model",    model)
counters:incr(provider .. "_today", 1, 0)
